De JSON à YAML

YAML (Yet Another Markup Language) est un autre format de représentation de données, axé sur la simplicité et lisibilité.

1. Liste

JSON
 [
  "one",
  "two",
  "three",
  "four"
]
YAML
- one
- two
- three
- four

1.1. Listes imbriquées

exemple 1

JSON
[
  "Taylor Swift",
  1989,
  ["Big Machine", "Republic"]
]
YAML
- Taylor Swift
- 1989
- - Big Machine
  - Republic

exemple 2

JSON
[
  [
    "Evermore",
    "Folklore",
    "Lover"
  ],
  [
    "I Am Easy to Find",
    "Sleep Well Beast",
    "Trouble Will Find Me"
  ]
]
YAML
- - Evermore
  - Folklore
  - Lover
- - I Am Easy to Find
  - Sleep Well Beast
  - Trouble Will Find Me

2. Objets

JSON
{
  "id": 1 ,
  "name": "Franck"
}
YAML
id: 1
name: Franck

2.1. Objets imbriqués

JSON
{
  "Taylor Swift": {
    "album_count": 9,
    "label": "Republic"
  },
  "The National": {
    "album_count": 8,
    "label": "4AD"
  }
}
YAML
Taylor Swift:
  album_count: 9
  label: Republic
The National:
  album_count: 8
  label: 4AD

3. Mixtes

3.1. Liste d’objets

JSON
[{
  "id": 1 ,
  "name": "Franck"
},
{
  "id": 2 ,
  "name": "John"
}]
YAML
- id: 1
  name: Franck
- id: 2
  name: John

3.2. Objets de listes

exemple 1

JSON
{
  "key1": [
    "value1",
    "value2",
    "value3",
    "value4",
    "value5"
  ]
}
YAML
key1:
  - value1
  - value2
  - value3
  - value4
  - value5

ou équivalent :

YAML
key1: [value1,value2,value3,value4,value5]

exemple 2

JSON
{
  "name": "Netlify Rebuild",
  "runs-on": "ubuntu-latest",
  "steps": [
    1,
    2,
    3
  ]
}
YAML
name: Netlify Rebuild
runs-on: ubuntu-latest
steps:
  - 1
  - 2
  - 3

3.3. Structures plus complexes

exemple 1

JSON
[{
  "id": 1 ,
  "nae": "Franck",
  "roles":["admin","hr"]
},
{
  "id": 2 ,
  "nae": "John",
    "roles":["admin","finance"]
}]
YAML
- id: 1
  nae: Franck
  roles:
  - admin
  - hr
- id: 2
  name: John
  roles:
  - admin
  - finance

exemple 2

JSON
{"data":
  [{
  "id": 1 ,
  "name": "Franck",
  "roles":["admin","hr"]
},
{
  "id": 2 ,
  "name": "John",
    "roles":["admin","finance"]

}] }

YAML
data:
- id: 1
  nae: Franck
  roles:
  - admin
  - hr
- id: 2
  nae: John
  roles:
  - admin
  - finance
